## 🧭 一、職涯起點與專業積累：建立端到端的技術視野

我畢業於銘傳大學資訊管理學系。在學期間，資管背景讓我建立起紮實的程式開發基礎，並學會從業務邏輯的角度出發，思考系統設計的合理性。在校期間，我經常在專案小組中擔任技術協調者的角色，負責釐清需求並轉化為可執行的架構設計，這份對技術細節的堅持，也成為我日後職涯的核心。

我的職涯首站始於虹堡/虹瑋科技。在那段期間，我擔任管理師（SA / Software Engineer），負責開發與建置企業內部核心系統，如 CRM、收銀平台及電子發票服務。這段經歷讓我深入接觸了從 ASP.NET 到 Spring Boot 的技術轉型，並具備了帶領前端團隊與跨海遠距協作的實戰經驗。特別是在導入 Service Mesh 與 Jenkins CI/CD 的過程中，我深刻體會到環境標準化與自動化部署對團隊效率的巨大提升。

## 🚀 二、現職突破與技術創新：推動 AI 驅動的工程革命

2021 年進入遠傳電信擔任專業工程師後，我的工作重心轉向企業級核心模組設計、微服務架構導入以及雲端原生平台的自動化運維。在這一階段，我不僅致力於功能的交付，更專注於思考如何透過技術手段，解決大型組織在系統開發中的效率瓶頸與工程負債。

我最顯著的專業貢獻在於將 AI 協作與 Spec-Driven Development (SDD) 流程成功落地於開發團隊中。透過導入 GitHub Spec-Kit，我將原本需耗時 4 小時的規格文件產出縮短至 30 分鐘（效率提升 88%），並將因理解落差造成的返工率從 25% 降至 10%。在 Infrastructure 方面，我建立的 Kubernetes 標準化部署流程，將原本 7 個步驟的手動作業轉化為自動化 Pipeline，使部署時間由 45 分鐘大幅降至 3 分鐘（效率提升 93%）。這些成果不僅是數據上的優化，更體現了我對「工程卓越」的執著追求。

## 🛡️ 三、競爭優勢：架構深度、安全意識與 AI 工具鏈整合

在長達八年的軟體開發生涯中，我磨練出以下三項核心競爭優勢：

1. 具備 AI 賦能的工程思維：我不僅能運用 GitHub Copilot 撰寫程式，更具備設計 AI 協作工作流的能力。我擅長利用 MCP (Model Context Protocol) 串接開發環境規格，確保 AI 生成內容與環境規格一致。這使我能以極高的效率完成高品質的程式開發與架構原型，並顯著降低人為失誤。
2. 嚴謹的資安與身份架構掌控力：我主導過企業級權限管理平台的後端架構設計，深度整合 Keycloak 與 SSO 登入流程，並設計出覆蓋超過 100 支 API 的權限映射機制。這項經驗讓我對身分認證、授權規則及資安合規有著極高的敏銳度，能為高安全性要求的系統提供可靠的防護基礎。
3. 全生命週期的開發實力：從需求訪談、API 契約設計（如 ui-contract.md 的推動）到 Kubernetes 雲端部署，我具備完整的全職責開發經驗。我熟悉 Java Spring Boot、Python FastAPI 以及 C# 等多種語言環境，並能有效整合 Redis、Consul 等中間件，確保系統在高負載環境下的穩定與高效。

## 🎯 四、工作動機：對「工程品質」與「技術嚴謹度」的渴望

我計劃離開現職，核心原因在於我對技術標準與系統品質的堅持。目前的工作型態較偏向專案交付導向，重點多放在如期完成任務；但我個人更重視系統品質、架構完整性與長期可維護性。

人壽產業的系統對穩定性、一致性與安全性有著極高的要求，這正是能讓我發揮專業、落實高品質工程流程的最佳平台。我希望尋找一個能同時兼顧交付效率與技術品質的工作環境，將我在 AI 自動化流程與微服務架構的經驗，應用於建立更為穩健且具備技術韌性的企業級系統中。

## 📈 五、未來五年職涯及學習規劃：成為架構領域的技術專家

展望未來五年，我為自己設定了階梯式的專業成長目標：

- 第 1-2 年（技術落地與流程優化）：我將專注於將現有的 AI 開發流程與雲端自動化經驗整合進新的工作團隊中。目標是透過更精確的規格設計與自動化檢測，降低系統建置初期的技術債，確保每一個模組的交付都符合高品質與高安全標準。
- 第 3-4 年（架構深度演進）：朝向資深系統架構師邁進。我計畫主導具備高度容錯性與可擴展性的系統規劃，並在零信任（Zero Trust）架構下持續優化身分認證平台，確保系統能應對日益複雜的網路環境與資安挑戰。

在學習規劃上，我將持續精進 Python 非同步開發與 Go 語言以應對高性能場景，並深入研究 AI-assisted Engineering Workflow，確保我的技術棧始終位於業界前端，能為企業提供最穩定的技術後盾。
